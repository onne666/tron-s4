
目标：基于你刚才的反馈，重新调整首页的信息节奏，让用户先“看懂支持什么 + 这套流程做什么 + 核心能力细节”，再在页面底部做主要转化。

1. 首页信息层级重构
- 把当前首屏主按钮从顶部 hero 区移走，不再一上来就强推连接。
- 顶部改成“品牌 + 标题 + 副标题 + 支持钱包 Logo 条 + 流程动画引导”。
- 将主 CTA「连接 Tron 钱包 / 立即检测我的钱包」移动到页面更靠下的位置，放在用户读完核心能力、信任背书之后，形成自然转化路径。
- 顶部保留一个弱化的辅助入口（如轻量跳转箭头或“了解检测流程”），避免首屏过早打断阅读。

2. 支持钱包展示改为 Logo 化
- 去掉当前 helperText 里的“支持 TronLink / ImToken / TokenPocket / Trust Wallet”纯文字表达。
- 在页面靠上的位置新增“支持钱包”横向 Logo 展示区，让用户第一眼就看到兼容范围。
- Logo 展示建议包含：TronLink、imToken、TokenPocket、Trust Wallet。
- 检测控制台区域不再重复大段钱包支持文案，只保留状态反馈、连接结果和错误提示。
- 若项目里没有现成素材，实施时可优先使用 SVG/简化品牌图形方案，确保视觉统一且移动端清晰。

3. 新增“检测流程动画区”
- 在 hero 下方或紧接支持钱包区，新增一个独立的“流程演示动画模块”。
- 用动态步骤流展示：连接钱包 → 授权检测 → 链上分析 → 输出结论。
- 这个模块不只是静态步骤条，而是加入自动轮播/高亮流动效果，例如：
  - 当前步骤发光
  - 连线扫描流动
  - 雷达/节点轻微动态
  - 文案自动切换
- 目标是让用户不用读很多字，也能秒懂这是一个“钱包安全检测流程”。

4. 核心检测能力模块增强为“可感知的能力卡片”
- 当前四个能力卡片从静态说明升级为带动画和细节提示的交互卡片。
- 每张卡片加入对应的微动效，让用户感知“黑名单检测 / 风险地址识别 / 资金来源分析 / 地址行为分析”具体在做什么。
- 推荐交互方式：
  - 卡片进入时依次浮现
  - hover / tap 时图标发光、边框增强
  - 卡片内增加动态示意元素（例如扫描线、节点连线、路径流向、脉冲点）
  - 可加入一句更具体的子说明或展开态内容
- 移动端优先，点击卡片也能看到能力细节，不能只依赖 hover。

5. 检测控制台模块内容优化
- 控制台顶部继续保留“连接钱包 → 授权 → 检测 → 结果”的专业感，但把“支持哪些钱包”的信息从这里前移到页面上方。
- 控制台自身聚焦在：
  - 钱包连接状态
  - 地址与余额
  - 授权提示
  - 检测中反馈
  - 错误与成功结果
- helperText 改成真正的流程反馈文案，不再承担宣传说明功能。

6. CTA 布局策略调整
- 在页面底部信任背书之后，新增一个更强的转化区块：
  - 标题：立即开始钱包安全检测
  - 简短说明：连接钱包后即可完成授权并进入链上分析
  - 主按钮：连接 Tron 钱包 / 立即检测我的钱包
- 如果希望进一步提升转化，也可以在页面最底部增加一个悬浮底栏按钮，但默认不建议首屏直接强推；优先先做“内容读完后的主 CTA”。

7. 动效与视觉实现方向
- 延续现有 Tron 红 + 深色科技风，不改整体品牌基调。
- 复用已有 framer-motion 与现有 CSS 动画体系，新增少量专用动画类即可，不需要推翻当前样式系统。
- 动画重点控制在“有吸引力但不干扰”：
  - 首屏流程动画持续但节奏平稳
  - 能力卡片微动效
  - 钱包 Logo 区轻微浮动/呼吸光效
- 保留 reduced-motion 兼容，避免移动端过度闪烁或性能压力。
